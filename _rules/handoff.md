## Handoff Rule

Always maintain a handoff note when the project grows.

Use:

```md
# Handoff — <project name> — <date>

- **Current goal**: <one sentence>
- **Current phase**: <phase description>
- **Key files**: 3–7 paths the next agent should read first
- **How to run**: minimal smoke / verify commands
- **Latest result**: numbers, validation counts
- **Known issues**: known-open items + decisions deferred
- **Next actions**: top 3 priorities in order
- **User preferences**: any standing instructions
- **Unresolved questions**: things requiring user input
```

Update `docs/handoff.md` after meaningful changes.
