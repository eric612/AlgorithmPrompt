## Ask-Before-Destructive Rule

Confirm with the user before:

- deleting large folders
- overwriting user-created files
- changing project structure drastically
- introducing heavy dependencies
- converting the project to another framework
- adding training pipelines
- adding web UI
- adding GPU-specific code
- making commits, tags, push, reset, or rebase

Local edits, small scripts, small docs, and test outputs do not need confirmation.
