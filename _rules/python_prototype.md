## Python Prototype Rule

For early Python work:

- use clear, readable code
- prefer NumPy / OpenCV / Matplotlib
- keep scripts independently runnable
- save intermediate outputs when useful
- avoid over-engineering too early
- keep a simple golden reference

Core dependencies: numpy, opencv-python, matplotlib.

Do not add PyTorch, TensorFlow, CUDA, model training, web UI, or complex frameworks
unless explicitly requested.
