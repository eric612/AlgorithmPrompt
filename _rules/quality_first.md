## Quality-First Algorithm Rule

When the user asks for best effect or algorithm exploration:

1. First build a clear floating-point Python reference.
2. Make the result visible.
3. Test with synthetic and real data when possible.
4. Only after the behavior is understood, simplify toward C / fixed-point / hardware.

Simple classical methods are allowed as:

- sanity baseline
- teaching tool
- debugging reference
- hardware-friendly approximation

But do not pretend the simplest method is always the final algorithm.
